Modal

콘텐츠를 레이어 위에 표시하는 다이얼로그 컴포넌트입니다. HTML 네이티브 <dialog> 요소를 기반으로 showModal() API를 활용하며, WAI-ARIA role="dialog" 패턴을 준수합니다. 포커스 트랩으로 모달 내부에서만 Tab이 이동하고, Esc 키로 닫을 수 있습니다. JavaScript 없이도 open 속성이 있는 <dialog> 내용이 페이지에 정적으로 표시됩니다.

Basic Modal

기본 모달입니다. data-modal-trigger 버튼이 클릭되면 dialog.showModal()을 호출하여 모달을 엽니다. Esc 키 또는 닫기 버튼으로 닫을 수 있으며, 배경 클릭으로도 닫힙니다. JavaScript 없이도 open 속성으로 정적 표시됩니다.

기본 모달

이것은 기본 모달 컴포넌트입니다. HTML 네이티브 <dialog> 요소를 사용하여 구현했습니다.

Esc 키를 누르거나 배경을 클릭하면 모달이 닫힙니다. Tab 키는 모달 내부에서만 이동합니다. 

<!-- 트리거 버튼: data-modal-target으로 dialog 연결 -->
<button
  type="button"
  data-modal-trigger
  data-modal-target="modal-basic"
  aria-haspopup="dialog"
>
  모달 열기
</button>

<!--
  dialog 요소:
  - open: JS 없을 때도 정적으로 내용 표시
  - role="dialog": WAI-ARIA 다이얼로그 역할
  - aria-modal="true": 모달임을 스크린 리더에 전달
  - aria-labelledby: h2 제목 요소와 연결
-->
<dialog
  id="modal-basic"
  role="dialog"
  aria-modal="true"
  aria-labelledby="modal-basic-title"
  open
>
  <div class="modal modal--md">
    <header class="modal__header">
      <h2 id="modal-basic-title" class="modal__title">기본 모달</h2>
      <button type="button" class="modal__close"
        data-modal-close aria-label="모달 닫기">
        <i data-lucide="x" aria-hidden="true"></i>
      </button>
    </header>
    <div class="modal__body">
      <p>모달 본문 내용입니다.</p>
    </div>
    <footer class="modal__footer">
      <button type="button" class="modal__btn modal__btn--cancel"
        data-modal-close>취소</button>
      <button type="button" class="modal__btn modal__btn--confirm"
        data-modal-close>확인</button>
    </footer>
  </div>
</dialog>
/* dialog 기본 스타일 */
dialog {
  border: none;
  padding: 0;
  background: transparent;
  max-width: 560px;
  width: 100%;
  margin: var(--spacing-xl) auto;
}

/* dialog::backdrop: showModal() 시 자동 생성되는 오버레이 */
dialog::backdrop {
  background-color: rgba(0, 0, 0, 0.5);
  animation: modal-overlay-in var(--transition-normal) ease forwards;
}

/* .modal: 실제 모달 박스 */
.modal {
  background-color: var(--color-bg);
  border-radius: var(--radius-lg);
  box-shadow: var(--shadow-xl);
  display: flex;
  flex-direction: column;
  overflow: hidden;
  animation: modal-in var(--transition-normal) ease forwards;
}

/* 크기 변형 */
.modal--sm { max-width: 400px; }
.modal--md { max-width: 560px; }
.modal--lg { max-width: 800px; }
// modal.js 자동 초기화: DOMContentLoaded 시 Modal.init() 호출
// 특정 컨테이너 범위만 초기화:
Modal.init(document.querySelector('.my-container'));

// 프로그래밍 방식으로 모달 열기:
const dialog = document.getElementById('modal-basic');
dialog.showModal(); // body scroll lock은 modal.js가 처리

// 닫기:
dialog.close();

Size Variants

.modal--sm, .modal--md, .modal--lg 클래스로 모달 너비를 조정합니다. 각각 400px, 560px, 800px의 최대 너비를 가집니다.

Small 모달

최대 너비 400px의 작은 모달입니다. 짧은 확인 메시지나 경고에 적합합니다.

Medium 모달

최대 너비 560px의 중간 모달입니다. 일반적인 정보 표시나 확인에 적합합니다.

다수의 텍스트나 간단한 폼 요소를 포함할 수 있습니다.

Large 모달

최대 너비 800px의 큰 모달입니다. 상세 정보, 이미지, 복잡한 내용을 표시할 때 사용합니다.

테이블, 차트, 긴 설명 텍스트 등 넓은 공간이 필요한 콘텐츠에 적합합니다.

모바일에서는 자동으로 하단 시트 형태로 전환됩니다.

 
<!-- Small: .modal--sm (max-width: 400px) -->
<dialog id="modal-sm" role="dialog"
  aria-modal="true" aria-labelledby="modal-sm-title">
  <div class="modal modal--sm">
    <header class="modal__header">
      <h2 id="modal-sm-title" class="modal__title">Small 모달</h2>
      <button type="button" class="modal__close"
        data-modal-close aria-label="모달 닫기">×</button>
    </header>
    <div class="modal__body"><p>내용</p></div>
    <footer class="modal__footer">
      <button class="modal__btn modal__btn--confirm"
        data-modal-close>확인</button>
    </footer>
  </div>
</dialog>

<!-- Medium: .modal--md (max-width: 560px) -->
<dialog id="modal-md" ...>
  <div class="modal modal--md">...</div>
</dialog>

<!-- Large: .modal--lg (max-width: 800px) -->
<dialog id="modal-lg" ...>
  <div class="modal modal--lg">...</div>
</dialog>
/* 크기 변형 클래스 */
.modal--sm { max-width: 400px; }
.modal--md { max-width: 560px; }
.modal--lg { max-width: 800px; }

/* 모바일에서는 모든 크기가 하단 시트로 */
@media (max-width: 767px) {
  .modal {
    max-width: 100%;
    border-radius: var(--radius-lg) var(--radius-lg) 0 0;
  }
}
// 크기 변형도 Modal.init()으로 자동 초기화
// 클래스만 변경하면 됨, 별도 설정 불필요
document.addEventListener('DOMContentLoaded', () => {
  Modal.init();
});

Form Modal

폼 입력 요소가 포함된 모달입니다. 모달이 열리면 첫 번째 입력 필드로 포커스가 자동 이동합니다. Tab 키로 입력 필드 간 이동하고, 마지막 요소에서 Tab 시 첫 번째 요소로 순환합니다.

프로필 편집

 
<!-- 폼 모달: 열리면 첫 번째 input으로 포커스 이동 -->
<dialog id="modal-form" role="dialog"
  aria-modal="true" aria-labelledby="modal-form-title">
  <div class="modal modal--md">
    <header class="modal__header">
      <h2 id="modal-form-title" class="modal__title">
        프로필 편집
      </h2>
      <button type="button" class="modal__close"
        data-modal-close aria-label="모달 닫기">×</button>
    </header>
    <div class="modal__body">
      <form id="profile-form">
        <label for="name">이름</label>
        <input type="text" id="name" name="name"
          placeholder="이름 입력">

        <label for="email">이메일</label>
        <input type="email" id="email" name="email"
          placeholder="이메일 입력">
      </form>
    </div>
    <footer class="modal__footer">
      <button type="button"
        class="modal__btn modal__btn--cancel"
        data-modal-close>취소</button>
      <button type="submit" form="profile-form"
        class="modal__btn modal__btn--confirm"
        data-modal-close>저장</button>
    </footer>
  </div>
</dialog>
// modal.js가 열릴 때 첫 번째 focusable 요소(input)로
// requestAnimationFrame을 통해 포커스 이동

// 포커스 트랩 동작 순서:
// input[name] → input[email] → textarea → 취소 → 저장 → 닫기(×)
// → 마지막에서 Tab 시 첫 번째(input)로 순환
// → 첫 번째에서 Shift+Tab 시 마지막(닫기)으로 순환

// 포커스 트랩이 탐색하는 요소 유형:
// 'a[href], button:not([disabled]), input:not([disabled]),
//  select:not([disabled]), textarea:not([disabled]),
//  [tabindex]:not([tabindex="-1"])'

Danger Confirmation

삭제 등 되돌릴 수 없는 작업 전 사용자 확인을 요구하는 패턴입니다. 취소 버튼에 autofocus 속성을 추가하면 실수로 확인 버튼을 누르는 것을 방지할 수 있습니다.

계정 삭제

이 작업은 되돌릴 수 없습니다. 계정과 모든 관련 데이터가 영구적으로 삭제됩니다.

계속하려면 취소 대신 삭제 버튼을 클릭하세요. 

<!-- Danger 확인 모달 -->
<dialog id="modal-danger" role="dialog"
  aria-modal="true"
  aria-labelledby="modal-danger-title"
  aria-describedby="modal-danger-desc">
  <div class="modal modal--sm">
    <header class="modal__header">
      <h2 id="modal-danger-title" class="modal__title"
        style="color: var(--color-danger);">
        계정 삭제
      </h2>
      <button type="button" class="modal__close"
        data-modal-close aria-label="모달 닫기">×</button>
    </header>
    <div class="modal__body">
      <p id="modal-danger-desc">
        이 작업은 <strong>되돌릴 수 없습니다.</strong>
        계정과 모든 관련 데이터가 영구적으로 삭제됩니다.
      </p>
    </div>
    <footer class="modal__footer">
      <!-- autofocus: 실수로 삭제 버튼 클릭 방지 -->
      <button type="button"
        class="modal__btn modal__btn--cancel"
        data-modal-close
        autofocus>취소</button>
      <button type="button"
        class="modal__btn modal__btn--danger"
        data-modal-close>삭제</button>
    </footer>
  </div>
</dialog>
// autofocus 속성이 있으면 showModal() 시 해당 요소로 포커스
// 취소 버튼에 autofocus → 실수로 삭제 버튼 클릭 방지

// 위험 동작에 대한 추가 보호 예시:
const dangerDialog = document.getElementById('modal-danger');
const deleteBtn = dangerDialog.querySelector('[data-modal-close]:last-child');

deleteBtn.addEventListener('click', () => {
  // 삭제 버튼 클릭 시 실행할 콜백
  console.log('계정 삭제 진행');
  // API 호출 등의 위험한 작업 수행
});

// alertdialog: 중요 경고에 사용하는 대안 role
// (현재는 role="dialog" 사용)
// role="alertdialog"로 변경하면 스크린 리더가
// 즉시 내용을 읽어줌
