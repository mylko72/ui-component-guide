Tooltip

요소에 마우스를 올리거나 포커스할 때 짧은 설명을 제공하는 컴포넌트입니다. WAI-ARIA tooltip 패턴을 준수하며 data-component="tooltip"으로 컨테이너를 지정하고, data-tooltip-trigger 버튼과 data-tooltip-content 패널을 aria-describedby로 연결합니다. JavaScript 없이도 CSS :hover와 :focus-within으로 툴팁이 표시됩니다.

Default Tooltip

기본 툴팁입니다. 버튼에 마우스를 올리거나 키보드 포커스를 주면 상단에 툴팁이 표시됩니다. JavaScript 없이도 CSS :hover / :focus-within으로 작동합니다.

짧은 설명 텍스트입니다
현재 작업을 저장합니다 (Ctrl+S) 
<!-- data-component="tooltip": 초기화 대상 컨테이너 -->
<div class="tooltip" data-component="tooltip" data-tooltip-placement="top">

  <!--
    data-tooltip-trigger: 트리거 식별자
    aria-describedby: 툴팁 ID (정적 마크업 — JS 없어도 스크린 리더 연결)
  -->
  <button
    type="button"
    data-tooltip-trigger
    aria-describedby="tooltip-1"
  >
    호버하거나 포커스하세요
  </button>

  <!--
    role="tooltip": 스크린 리더에 역할 전달
    id: aria-describedby와 반드시 일치
  -->
  <div
    id="tooltip-1"
    class="tooltip__content"
    data-tooltip-content
    role="tooltip"
  >
    짧은 설명 텍스트입니다
  </div>

</div>
/* JS 없을 때: :hover/:focus-within 폴백 */
[data-component="tooltip"]:not([data-js-loaded]):hover [data-tooltip-content],
[data-component="tooltip"]:not([data-js-loaded]):focus-within [data-tooltip-content] {
  opacity: 1;
  visibility: visible;
}

/* JS 활성화 후: 클래스 기반 제어 */
[data-component="tooltip"][data-js-loaded] [data-tooltip-content].tooltip--visible {
  opacity: 1;
  visibility: visible;
}

/* 포커스 가시성 */
[data-tooltip-trigger]:focus-visible {
  outline: 2px solid var(--color-primary);
  outline-offset: 2px;
}
// DOMContentLoaded 시 자동 초기화
document.addEventListener('DOMContentLoaded', () => {
  Tooltip.init(); // 페이지의 모든 [data-component="tooltip"] 초기화
});

// 특정 컨테이너 범위만 초기화:
Tooltip.init(document.querySelector('.my-container'));

Placements

data-tooltip-placement 속성으로 툴팁의 표시 방향을 지정합니다. top(기본), bottom, left, right 4가지 방향을 지원합니다.

위쪽에 표시됩니다
아래쪽에 표시됩니다
왼쪽에 표시됩니다
오른쪽에 표시됩니다 
<!-- top (기본) -->
<div class="tooltip" data-component="tooltip" data-tooltip-placement="top">
  <button type="button" data-tooltip-trigger aria-describedby="tt-top">Top</button>
  <div id="tt-top" class="tooltip__content" data-tooltip-content role="tooltip">
    위쪽에 표시됩니다
  </div>
</div>

<!-- bottom -->
<div class="tooltip" data-component="tooltip" data-tooltip-placement="bottom">
  <button type="button" data-tooltip-trigger aria-describedby="tt-bottom">Bottom</button>
  <div id="tt-bottom" class="tooltip__content" data-tooltip-content role="tooltip">
    아래쪽에 표시됩니다
  </div>
</div>

<!-- left -->
<div class="tooltip" data-component="tooltip" data-tooltip-placement="left">
  <button type="button" data-tooltip-trigger aria-describedby="tt-left">Left</button>
  <div id="tt-left" class="tooltip__content" data-tooltip-content role="tooltip">
    왼쪽에 표시됩니다
  </div>
</div>

<!-- right -->
<div class="tooltip" data-component="tooltip" data-tooltip-placement="right">
  <button type="button" data-tooltip-trigger aria-describedby="tt-right">Right</button>
  <div id="tt-right" class="tooltip__content" data-tooltip-content role="tooltip">
    오른쪽에 표시됩니다
  </div>
</div>
/* bottom: 아래쪽 배치 */
[data-tooltip-placement="bottom"] [data-tooltip-content] {
  top: calc(100% + 8px);
  bottom: auto;
  left: 50%;
  transform: translateX(-50%);
}

/* left: 왼쪽 배치 */
[data-tooltip-placement="left"] [data-tooltip-content] {
  right: calc(100% + 8px);
  left: auto;
  top: 50%;
  transform: translateY(-50%);
}

/* right: 오른쪽 배치 */
[data-tooltip-placement="right"] [data-tooltip-content] {
  left: calc(100% + 8px);
  top: 50%;
  transform: translateY(-50%);
}
// data-tooltip-placement 값에 따라 CSS가 자동으로 방향 처리
// 별도 JS 설정 불필요

// 동적으로 방향 변경 예시 (재초기화 없이 가능):
const tooltipEl = document.querySelector('[data-component="tooltip"]');
tooltipEl.dataset.tooltipPlacement = 'bottom';

Icon Trigger

아이콘만 있는 버튼에 툴팁을 연결할 때는 aria-label로 버튼의 목적을 먼저 전달하고, aria-describedby로 툴팁의 추가 설명을 연결합니다.

환경설정 열기
이 항목에 대한 자세한 설명을 확인합니다
이 항목을 삭제합니다 (되돌릴 수 없음)
클립보드에 복사 
<!--
  아이콘 버튼에 툴팁 연결:
  - aria-label: 버튼 목적 전달 (필수)
  - aria-describedby: 추가 설명 연결
-->
<div class="tooltip" data-component="tooltip" data-tooltip-placement="top">
  <button
    type="button"
    data-tooltip-trigger
    aria-label="설정"
    aria-describedby="tooltip-settings"
  >
    <i data-lucide="settings" aria-hidden="true"></i>
  </button>
  <div
    id="tooltip-settings"
    class="tooltip__content"
    data-tooltip-content
    role="tooltip"
  >
    환경설정 열기
  </div>
</div>
/* 아이콘 버튼 포커스 가시성 */
[data-tooltip-trigger]:focus-visible {
  outline: 2px solid var(--color-primary);
  outline-offset: 2px;
  border-radius: var(--radius-sm);
}
// 아이콘 버튼 toolip은 별도 JS 설정 불필요
// Tooltip.init()이 자동으로 감지하여 초기화

// 접근성 핵심:
// 1. aria-label → 버튼의 목적 (스크린 리더가 버튼 역할을 읽음)
// 2. aria-describedby → 툴팁의 추가 설명 (선택적 상세 설명)
// 스크린 리더: "설정 버튼, 환경설정 열기" 와 같이 읽음

Disabled State

disabled 버튼은 마우스/포커스 이벤트를 받지 못하므로 툴팁이 동작하지 않습니다. 비활성 버튼에 툴팁을 표시해야 할 경우, data-tooltip-trigger를 래퍼 <span>에 적용하고 버튼에 aria-disabled="true"를 사용하는 패턴을 권장합니다.

이 툴팁은 표시되지 않습니다
권한이 없어 이 기능을 사용할 수 없습니다 
<!-- 패턴 A: disabled 버튼 — 툴팁 동작 안 함 -->
<div class="tooltip" data-component="tooltip">
  <button type="button" data-tooltip-trigger disabled>
    disabled 버튼
  </button>
  <div data-tooltip-content role="tooltip">
    이 툴팁은 표시되지 않습니다
  </div>
</div>

<!-- 패턴 B: aria-disabled + 래퍼 span (권장) -->
<div class="tooltip" data-component="tooltip">
  <span
    data-tooltip-trigger
    tabindex="0"
    style="display:inline-flex;"
  >
    <button
      type="button"
      aria-disabled="true"
      style="pointer-events:none;opacity:0.4;"
    >
      비활성 버튼
    </button>
  </span>
  <div data-tooltip-content role="tooltip">
    권한이 없어 이 기능을 사용할 수 없습니다
  </div>
</div>
/* aria-disabled 버튼 시각적 비활성 표현 */
[aria-disabled="true"] {
  opacity: 0.4;
  cursor: not-allowed;
  pointer-events: none;
}

/* 래퍼 span 포커스 가시성 */
span[data-tooltip-trigger]:focus-visible {
  outline: 2px solid var(--color-primary);
  outline-offset: 2px;
  border-radius: var(--radius-sm);
}
// 핵심 차이:
// disabled: 이벤트 수신 불가 → 툴팁 불가
// aria-disabled="true": 이벤트 수신 가능 → 툴팁 가능

// 래퍼 span 패턴으로 비활성 버튼 툴팁 구현:
// 1. span[data-tooltip-trigger][tabindex="0"]이 이벤트 수신
// 2. 버튼에 pointer-events:none으로 클릭 차단
// 3. 버튼에 aria-disabled="true"로 스크린 리더에 상태 전달
